========================================
4. BACKEND API INTEGRATION
========================================
Endpoints used:
- GET /api/pets/nearby – fetch other users' pets
- POST /api/pet-matches – send match request
- GET /api/pet-matches/incoming – fetch incoming requests
- PUT /api/pet-matches/:id – accept/reject request

========================================
5. USER FLOW
========================================
1. User adds a pet (My Pets tab)
2. User navigates to Pet Matches tab
3. System checks if user has a pet
4. Displays nearby pets from other users
5. User clicks "Request Play Date"
6. Request saved to database with 'pending' status
7. Target pet owner receives request
8. Owner can accept or decline
9. Status updates in real-time
